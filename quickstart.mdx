---
title: "Get started with Lynx"
description: "Run Lynx locally with Docker Compose, connect your GitHub account, add an LLM API key, and get your first AI code review in under ten minutes."
---

This guide walks you through running Lynx on your local machine using the development Docker Compose configuration. At the end you will have a working Lynx instance connected to GitHub, with an LLM API key configured and ready to review pull requests.

<Info>
  This quickstart uses `docker-compose.dev.yml`, which runs Lynx over HTTP on localhost. For a production deployment with HTTPS, see the [Deployment](/deployment) guide.
</Info>

## Prerequisites

- [Docker](https://docs.docker.com/get-docker/) and Docker Compose installed
- A GitHub account (GitLab is also supported — see [GitLab configuration](/configuration/gitlab))
- An API key for at least one LLM provider: OpenAI, Gemini, or Claude

<Steps>
  <Step title="Clone the repository">
    Clone the Lynx repository and navigate into it.

    <CodeGroup>

    ```bash Terminal
    git clone https://github.com/tarek-gritli/lynx.git
    cd lynx
    ```

    </CodeGroup>
  </Step>

  <Step title="Copy the environment file">
    Copy the example environment file. You will fill in the required values in the next step.

    <CodeGroup>

    ```bash Terminal
    cp .env.example .env
    ```

    </CodeGroup>
  </Step>

  <Step title="Configure your environment">
    Open `.env` in your editor and fill in the required variables for local development.

    ```bash .env
    # GitHub App credentials (see /configuration/github-app to create one)
    GITHUB_APP_ID=
    GITHUB_PRIVATE_KEY_PATH=/path/to/your/github-app-key.pem
    GITHUB_WEBHOOK_SECRET=your_webhook_secret
    GITHUB_CLIENT_ID=your_github_client_id
    GITHUB_CLIENT_SECRET=your_github_client_secret

    # Security — generate strong random values for these
    JWT_SECRET=your_jwt_secret
    ENCRYPTION_KEY=your_fernet_encryption_key

    # URLs for local development
    BACKEND_URL=http://localhost:8000
    FRONTEND_URL=http://localhost:5173
    ```

    <Warning>
      `JWT_SECRET` and `ENCRYPTION_KEY` must be strong random values even for local development. Generate `JWT_SECRET` with `openssl rand -hex 32`. Generate `ENCRYPTION_KEY` with `python3 -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"` — it must be a valid Fernet key.
    </Warning>

    | Variable | Description |
    | --- | --- |
    | `GITHUB_APP_ID` | Numeric ID shown on your GitHub App's settings page |
    | `GITHUB_PRIVATE_KEY_PATH` | Absolute path to the `.pem` private key downloaded from GitHub |
    | `GITHUB_WEBHOOK_SECRET` | Secret you set when creating the GitHub App |
    | `GITHUB_CLIENT_ID` | OAuth client ID from the GitHub App settings |
    | `GITHUB_CLIENT_SECRET` | OAuth client secret from the GitHub App settings |
    | `JWT_SECRET` | Random secret used to sign session tokens |
    | `ENCRYPTION_KEY` | Fernet key used to encrypt stored LLM API keys |
    | `BACKEND_URL` | Public URL of the backend — use `http://localhost:8000` locally |
    | `FRONTEND_URL` | Public URL of the frontend — use `http://localhost:5173` locally |
  </Step>

  <Step title="Start the services">
    Start the backend, frontend, and PostgreSQL database using the development Compose file.

    <CodeGroup>

    ```bash Terminal
    docker-compose -f docker-compose.dev.yml up -d
    ```

    </CodeGroup>

    Docker will build the images on the first run — this takes a minute or two. Once all containers are healthy, three services are running: `lynx-backend`, `lynx-frontend`, and `lynx-db`.
  </Step>

  <Step title="Open Lynx in your browser">
    Once the services are up, open the frontend in your browser:

    - **Frontend:** [http://localhost:5173](http://localhost:5173)
    - **Backend API:** [http://localhost:8000/api/v1](http://localhost:8000/api/v1)
  </Step>

  <Step title="Log in with GitHub">
    Click **Log in with GitHub** on the Lynx home page. You will be redirected to GitHub to authorize the OAuth application, then returned to the Lynx dashboard.
  </Step>

  <Step title="Add your LLM API key">
    In the Lynx dashboard, navigate to **Settings** and add an API key for at least one LLM provider — OpenAI, Gemini, or Claude. Keys are encrypted with Fernet before being stored in the database.

    <Note>
      You can add keys for multiple providers at the same time. When more than one key is configured, Lynx runs all providers in parallel and posts each review as a separate comment.
    </Note>
  </Step>

  <Step title="Install the GitHub App on a repository">
    Go to **Settings → GitHub App** in the dashboard and install the Lynx GitHub App on at least one repository. This grants Lynx the permissions it needs to read diffs and post review comments.

    For detailed instructions on creating and installing the GitHub App, see the [GitHub App configuration](/configuration/github-app) guide.
  </Step>

  <Step title="Open a pull request">
    Open a pull request on any repository where the Lynx GitHub App is installed. Within a few seconds, Lynx will post an AI-generated review comment on the PR.

    You can also trigger a review on an existing PR by posting `/review` as a comment.
  </Step>
</Steps>
