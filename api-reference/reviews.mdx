---
title: 'Reviews and statistics API endpoints'
description: 'List, filter, and inspect AI code reviews. Query all-time usage statistics and period-over-period changes with a configurable rolling window.'
---

The reviews and statistics endpoints let you query the history of AI code reviews that Lynx has performed on your behalf, retrieve the full content of any individual review, and fetch aggregate usage metrics. All endpoints are scoped to the authenticated user — you can only access your own reviews.

<Note>
  The route prefix for these endpoints is `/usage`, not `/dashboard`. The full path for listing reviews is `GET /api/v1/usage/reviews`.
</Note>

---

## GET /usage/reviews

Returns a paginated, filterable list of code reviews for the authenticated user, ordered by most recent first.

**Authentication required:** yes

```bash
curl "https://yourdomain.com/api/v1/usage/reviews?page=1&limit=10&status=success" \
  -b "lynx_session=<your-session-cookie>"
```

### Query parameters

<ParamField query="provider" type="string">
  Filter by LLM provider. Accepted values: `openai`, `gemini`, `anthropic`.
</ParamField>

<ParamField query="model" type="string">
  Filter by model name (e.g., `gpt-4o`, `claude-3-5-sonnet-20241022`).
</ParamField>

<ParamField query="repo_name" type="string">
  Filter by repository full name (e.g., `owner/repo`).
</ParamField>

<ParamField query="start_date" type="string">
  Filter reviews created at or after this timestamp. ISO 8601 format (e.g., `2024-01-01T00:00:00Z`).
</ParamField>

<ParamField query="end_date" type="string">
  Filter reviews created at or before this timestamp. ISO 8601 format.
</ParamField>

<ParamField query="status" type="string">
  Filter by review status. Accepted values: `pending`, `success`, `failed`.
</ParamField>

<ParamField query="page" type="integer" default="1">
  Page number. Minimum: `1`.
</ParamField>

<ParamField query="limit" type="integer" default="10">
  Number of results per page. Minimum: `1`, maximum: `100`.
</ParamField>

### Response

```json
{
  "reviews": [
    {
      "id": 42,
      "user_id": 1,
      "provider": "openai",
      "model": "gpt-4o",
      "repo_name": "owner/my-repo",
      "pr_number": 17,
      "pr_url": "https://github.com/owner/my-repo/pull/17",
      "status": "success",
      "error_message": null,
      "prompt_tokens": 1200,
      "completion_tokens": 350,
      "total_tokens": 1550,
      "cost": 0.009300,
      "review_text": "- 🔴 CRITICAL: SQL injection risk in ...",
      "created_at": "2024-06-01T14:23:11+00:00"
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 10,
    "total": 1,
    "pages": 1
  },
  "filters": {
    "status": "success"
  }
}
```

<ResponseField name="reviews" type="object[]">
  Array of review objects. See fields below.

  <Expandable title="Review object fields">
    <ResponseField name="id" type="integer">
      Unique review ID.
    </ResponseField>
    <ResponseField name="user_id" type="integer">
      ID of the user who owns this review.
    </ResponseField>
    <ResponseField name="provider" type="string">
      LLM provider used for the review (`openai`, `gemini`, or `anthropic`).
    </ResponseField>
    <ResponseField name="model" type="string">
      Specific model used (e.g., `gpt-4o`).
    </ResponseField>
    <ResponseField name="repo_name" type="string">
      Full repository name in `owner/repo` format.
    </ResponseField>
    <ResponseField name="pr_number" type="integer">
      Pull request or merge request number.
    </ResponseField>
    <ResponseField name="pr_url" type="string">
      Direct URL to the pull request or merge request.
    </ResponseField>
    <ResponseField name="status" type="string">
      Review status: `pending`, `success`, or `failed`.
    </ResponseField>
    <ResponseField name="error_message" type="string">
      Error details if the review failed. `null` otherwise.
    </ResponseField>
    <ResponseField name="prompt_tokens" type="integer">
      Number of tokens in the prompt sent to the LLM.
    </ResponseField>
    <ResponseField name="completion_tokens" type="integer">
      Number of tokens in the LLM's response.
    </ResponseField>
    <ResponseField name="total_tokens" type="integer">
      Sum of prompt and completion tokens.
    </ResponseField>
    <ResponseField name="cost" type="number">
      Estimated cost in USD for this review.
    </ResponseField>
    <ResponseField name="review_text" type="string">
      The full review output posted as a PR comment. `null` if the review is still pending or failed.
    </ResponseField>
    <ResponseField name="created_at" type="string">
      ISO 8601 timestamp of when the review was created.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="pagination" type="object">
  Pagination metadata.

  <Expandable title="Pagination fields">
    <ResponseField name="page" type="integer">
      Current page number.
    </ResponseField>
    <ResponseField name="limit" type="integer">
      Number of items per page.
    </ResponseField>
    <ResponseField name="total" type="integer">
      Total number of matching reviews across all pages.
    </ResponseField>
    <ResponseField name="pages" type="integer">
      Total number of pages.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="filters" type="object">
  The active filters that were applied to this query. Only includes filters that were provided.
</ResponseField>

## GET /usage/reviews/{review_id}

Returns the full details of a single review. Returns `404` if the review does not exist or belongs to a different user.

**Authentication required:** yes

```bash
curl https://yourdomain.com/api/v1/usage/reviews/42 \
  -b "lynx_session=<your-session-cookie>"
```

### Path parameters

<ParamField path="review_id" type="integer" required>
  The numeric ID of the review to retrieve.
</ParamField>

### Response

Returns a single review object with the same fields as listed in the reviews array above.

```json
{
  "id": 42,
  "user_id": 1,
  "provider": "openai",
  "model": "gpt-4o",
  "repo_name": "owner/my-repo",
  "pr_number": 17,
  "pr_url": "https://github.com/owner/my-repo/pull/17",
  "status": "success",
  "error_message": null,
  "prompt_tokens": 1200,
  "completion_tokens": 350,
  "total_tokens": 1550,
  "cost": 0.009300,
  "review_text": "- 🔴 CRITICAL: ...",
  "created_at": "2024-06-01T14:23:11+00:00"
}
```

## GET /usage/stats

Returns all-time usage totals and period-over-period changes for the authenticated user. The `window_days` parameter controls the size of the rolling comparison window.

**Authentication required:** yes

```bash
curl "https://yourdomain.com/api/v1/usage/stats?window_days=7" \
  -b "lynx_session=<your-session-cookie>"
```

### Query parameters

<ParamField query="window_days" type="integer" default="7">
  Length of the rolling comparison window in days. The current window is the last `N` days; the previous window is the `N` days before that. Minimum: `1`, maximum: `90`.
</ParamField>

### Response

```json
{
  "total_reviews": 156,
  "total_tokens": 238400,
  "total_cost": 1.430400,
  "successful_reviews": 149,
  "failed_reviews": 7,
  "period": {
    "window_days": 7,
    "current": {
      "start": "2024-05-25T14:23:11+00:00",
      "end": "2024-06-01T14:23:11+00:00"
    },
    "previous": {
      "start": "2024-05-18T14:23:11+00:00",
      "end": "2024-05-25T14:23:11+00:00"
    }
  },
  "period_counts": {
    "current": {
      "total_reviews": 24,
      "successful_reviews": 23,
      "failed_reviews": 1,
      "total_tokens": 36800,
      "total_cost": 0.220800
    },
    "previous": {
      "total_reviews": 18,
      "successful_reviews": 17,
      "failed_reviews": 1,
      "total_tokens": 27600,
      "total_cost": 0.165600
    }
  },
  "changes": {
    "total_reviews": { "percent": 33, "direction": "up" },
    "successful_reviews": { "percent": 35, "direction": "up" },
    "failed_reviews": { "percent": 0, "direction": "up" },
    "total_tokens": { "percent": 33, "direction": "up" },
    "total_cost": { "percent": 33, "direction": "up" }
  }
}
```

<ResponseField name="total_reviews" type="integer">
  All-time total number of reviews for this user.
</ResponseField>

<ResponseField name="total_tokens" type="integer">
  All-time total token usage across all reviews.
</ResponseField>

<ResponseField name="total_cost" type="number">
  All-time total estimated cost in USD, rounded to 6 decimal places.
</ResponseField>

<ResponseField name="successful_reviews" type="integer">
  All-time count of reviews with `status = success`.
</ResponseField>

<ResponseField name="failed_reviews" type="integer">
  All-time count of reviews with `status = failed`.
</ResponseField>

<ResponseField name="period" type="object">
  Describes the time windows used for period comparisons.

  <Expandable title="Period fields">
    <ResponseField name="window_days" type="integer">
      The requested rolling window size in days.
    </ResponseField>
    <ResponseField name="current" type="object">
      Start and end ISO 8601 timestamps for the current period.
    </ResponseField>
    <ResponseField name="previous" type="object">
      Start and end ISO 8601 timestamps for the previous period (immediately preceding the current window).
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="period_counts" type="object">
  Review counts and totals broken down for the current and previous periods.

  <Expandable title="Period count fields">
    <ResponseField name="current" type="object">
      Counts for `total_reviews`, `successful_reviews`, `failed_reviews`, `total_tokens`, and `total_cost` in the current window.
    </ResponseField>
    <ResponseField name="previous" type="object">
      Same fields as `current` but for the preceding window.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="changes" type="object">
  Percentage change from the previous period to the current period for each metric.

  <Expandable title="Change object fields">
    <ResponseField name="total_reviews" type="object">
      <Expandable title="properties">
        <ResponseField name="percent" type="integer">
          Absolute percentage change (always non-negative).
        </ResponseField>
        <ResponseField name="direction" type="string">
          `"up"` if the current period is equal to or greater than the previous, `"down"` otherwise.
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="successful_reviews" type="object">
      Same structure as `total_reviews`.
    </ResponseField>
    <ResponseField name="failed_reviews" type="object">
      Same structure as `total_reviews`.
    </ResponseField>
    <ResponseField name="total_tokens" type="object">
      Same structure as `total_reviews`.
    </ResponseField>
    <ResponseField name="total_cost" type="object">
      Same structure as `total_reviews`.
    </ResponseField>
  </Expandable>
</ResponseField>
