---
title: 'Review templates REST API endpoints'
description: 'Create, update, delete, and manage custom review prompt templates. Set a default template or fall back to the built-in system prompt.'
---

Templates let you customize the prompt Lynx sends to the LLM when reviewing a pull request. You can create multiple templates per user and designate one as the default. If no default is set, Lynx uses the built-in system prompt.

Every template content string must include the `{diff}` placeholder. Lynx replaces this at review time with the actual pull request diff.

---

## GET /templates

Returns all templates belonging to the authenticated user. Results are ordered with the default template first, then by most recently created.

**Authentication required:** yes

```bash
curl https://yourdomain.com/api/v1/templates \
  -b "lynx_session=<your-session-cookie>"
```

### Response

```json
[
  {
    "id": 3,
    "name": "Security focus",
    "content": "Review this diff for security vulnerabilities only.\n\n{diff}",
    "is_default": true,
    "created_at": "2024-05-20T09:00:00+00:00",
    "updated_at": "2024-05-21T11:30:00+00:00"
  }
]
```

<ResponseField name="id" type="integer">
  Unique template ID.
</ResponseField>

<ResponseField name="name" type="string">
  Template display name. Unique per user.
</ResponseField>

<ResponseField name="content" type="string">
  Full prompt content. Always contains `{diff}`.
</ResponseField>

<ResponseField name="is_default" type="boolean">
  Whether this is the user's currently active default template.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of creation.
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp of the most recent update.
</ResponseField>

## GET /templates/default

Returns the user's default template. If the user has not set a default, returns the built-in system prompt.

**Authentication required:** yes

```bash
curl https://yourdomain.com/api/v1/templates/default \
  -b "lynx_session=<your-session-cookie>"
```

### Response — user default

```json
{
  "id": 3,
  "name": "Security focus",
  "content": "Review this diff for security vulnerabilities only.\n\n{diff}",
  "is_default": true,
  "created_at": "2024-05-20T09:00:00+00:00",
  "updated_at": "2024-05-21T11:30:00+00:00"
}
```

### Response — system default (no user default set)

When no user default is configured, the response uses `id: null` and the built-in prompt:

```json
{
  "id": null,
  "name": "System Default",
  "content": "...",
  "is_default": true,
  "created_at": null,
  "updated_at": null
}
```

The full content of the system default prompt is:

```text
You are a code reviewer. Analyze this pull request diff and provide a concise review.

Focus on:
1. Critical bugs and logic errors
2. Security vulnerabilities (SQL injection, XSS, auth issues)
3. Performance problems (N+1 queries, inefficient algorithms)
4. Code smells that will cause maintainability issues

Be direct and concise. Only mention serious issues. Don't comment on style or formatting.

Format your response as a markdown list with clear severity indicators:
- 🔴 CRITICAL: Must fix before merge
- 🟡 WARNING: Should fix soon
- 🔵 SUGGESTION: Consider improving

Here's the diff:

{diff}

Provide your review:
```

## POST /templates

Creates a new template for the authenticated user.

**Authentication required:** yes

```bash
curl -X POST https://yourdomain.com/api/v1/templates \
  -b "lynx_session=<your-session-cookie>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Security focus",
    "content": "Review this diff for security issues only.\n\n{diff}\n\nList findings by severity."
  }'
```

### Request body

<ParamField body="name" type="string" required>
  A display name for the template. Must be unique per user. Returns `400` if a template with this name already exists.
</ParamField>

<ParamField body="content" type="string" required>
  The full prompt content. Must contain `{diff}` — returns `400` if the placeholder is missing. Lynx replaces `{diff}` with the pull request diff at review time.
</ParamField>

### Response

Returns the newly created template object:

```json
{
  "id": 5,
  "name": "Security focus",
  "content": "Review this diff for security issues only.\n\n{diff}\n\nList findings by severity.",
  "is_default": false,
  "created_at": "2024-06-01T12:00:00+00:00",
  "updated_at": "2024-06-01T12:00:00+00:00"
}
```

## PATCH /templates/{template_id}

Updates the name, content, or both for an existing template. Both fields are optional — omit a field to leave it unchanged.

**Authentication required:** yes

```bash
curl -X PATCH https://yourdomain.com/api/v1/templates/5 \
  -b "lynx_session=<your-session-cookie>" \
  -H "Content-Type: application/json" \
  -d '{"name": "Security & performance"}'
```

### Path parameters

<ParamField path="template_id" type="integer" required>
  ID of the template to update.
</ParamField>

### Request body

<ParamField body="name" type="string">
  New name for the template. Must be unique per user. Returns `400` if another template already uses this name.
</ParamField>

<ParamField body="content" type="string">
  New content for the template. Must include `{diff}` — returns `400` if missing.
</ParamField>

### Response

Returns the updated template object with the same fields as `POST /templates`.

Returns `404` if the template does not exist or belongs to a different user.

## DELETE /templates/{template_id}

Deletes a template. Returns `404` if the template does not exist or belongs to a different user.

**Authentication required:** yes

```bash
curl -X DELETE https://yourdomain.com/api/v1/templates/5 \
  -b "lynx_session=<your-session-cookie>"
```

### Path parameters

<ParamField path="template_id" type="integer" required>
  ID of the template to delete.
</ParamField>

### Response

```json
{
  "status": "success"
}
```

---

## POST /templates/set-default

Marks a template as the default for the authenticated user. Any previously set default is unset automatically.

**Authentication required:** yes

```bash
curl -X POST https://yourdomain.com/api/v1/templates/set-default \
  -b "lynx_session=<your-session-cookie>" \
  -H "Content-Type: application/json" \
  -d '{"template_id": 3}'
```

### Request body

<ParamField body="template_id" type="integer" required>
  ID of the template to set as default. Must belong to the authenticated user.
</ParamField>

### Response

```json
{
  "status": "success",
  "template_id": 3
}
```

<ResponseField name="status" type="string">
  Always `"success"`.
</ResponseField>

<ResponseField name="template_id" type="integer">
  The ID of the template that is now set as default.
</ResponseField>

Returns `404` if the template does not exist or belongs to a different user.

---

## POST /templates/reset-default

Removes the user's default template selection, reverting to the built-in system prompt.

**Authentication required:** yes

```bash
curl -X POST https://yourdomain.com/api/v1/templates/reset-default \
  -b "lynx_session=<your-session-cookie>"
```

No request body is required.

### Response

```json
{
  "status": "success"
}
```
